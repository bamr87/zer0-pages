# SCHEMA — images

> Site imagery — identity art referenced from _config.yml, plus subdirectories for avatars, docs screenshots, and generated preview banners.

## Structure

| entry | kind | purpose | rules |
|---|---|---|---|
| `authors/` | dir | Author avatar images keyed from _data/authors.yml | |
| `docs/` | dir | Screenshots embedded in _docs/ pages | |
| `previews/` | dir | AI preview banners generated by `bundle exec jekyll preview-images` (zer0-image-generator gem) — never hand-edit | generated |
| `quickstart/` | dir | Screenshots for the getting-started quick-start guide | |
| `favicon_gpt_computer_retro.png` | file | Fallback card/OG teaser image (_config.yml `teaser`) | |
| `gravatar-small.png` | file | Navbar branding logo and default author avatar (_config.yml `logo`) | |
| `info-banner-mountain-wizard.png` | file | Intro hero fallback banner (_config.yml `info_banner`) | |
| `wizard-on-journey.png` | file | Wizard illustration embedded by showcase posts | |

## Placement

- Post/doc banner images are generated into `previews/` — regenerate, don't hand-place.
- New docs screenshots → `docs/<area>/` matching the doc page that embeds them.
